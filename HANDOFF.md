## 🎯 현재 구현된 기능

### 1. 기본 UI
- **역할 선택 탭**: 탱커 / 딜러 / 힐러 (상단)
- **⚙️ 우선순위 편집 패널**: 10개 카테고리(공격/방어/치명/...)를 드롭다운으로 순위 지정
  - 같은 순위끼리 묶이면 동일 가중치
  - "제외" 선택 시 그 외(최저 가중치)로 처리
  - 역할별 localStorage 저장
  - 비고란: **작성자 추천 우선순위** 참고 표시

### 2. 최종 스탯 패널 (🔢)
- 8쌍 페어 스탯 입력: `최종값 × 보너스%` → 기본값 자동 계산 (`최종 ÷ 보너스%/100`)
  - 공격/방어/치명/치명저항/명중/관통/회피/HP
- 피해보너스/피해감소는 현재 % 단독 입력
- **🎯 목표 비율 입력**: 치명확률(기본 80%), 명중비율(25%), 관통비율(30%)
- "공격 기준 자동 채움" 버튼: 공격값 기준으로 관통/명중/치명 자동 계산
- 현재 비율 vs 목표 비율 색상 표시

### 3. 🛡️ 현재 장착 엠블럼 패널
- 그룹별(기교/지혜/활력) 4 슬롯 × 드롭다운 2개
  - 엠블럼 이름 + 급수 선택
  - 저장 포맷: `"광기 흑룡#4"` (이름#급수)
  - 레거시 데이터(급수 없음) 자동 마이그레이션

### 4. 💥 엠블럼 데이터 구조 (v2)
```js
{
  name: "광기 흑룡",
  unique: true,
  s1: "공격", s2: "피해보너스",
  grades: [null, null, G("+340","+7%"), G("+460","+10.5%"), G("+650","+13%")]
  // 인덱스 0~4 = 1급~5급. null이면 해당 급수 없음.
}
```
- 총 60개: 기교 18 + 지혜 24 + 활력 18
- 드랍 유형:
  - 1급 드랍 (최대 3급): 슬라임/고브/창무쉬/예티 계열 (12개 신규)
  - 2급 드랍 (최대 4급): 고유/비고유 혼재
  - 3급 드랍 (최대 5급): 상위 엠블럼

### 5. 🎯 결과보기 버튼
- 설정 변경 즉시 계산 안 됨 (성능)
- 버튼 클릭 시 최적 엠블럼 계산
- 설정 변경 시 버튼이 주황색 펄스 (⚠️ 설정 변경됨)
- 계산 완료 시 파란색 + ✓ 최신 결과

### 6. 🚫 급수 제외 기능
- 결과의 각 엠블럼 카드에 `🚫 N급 제외` 버튼
- 클릭 시 해당 (엠블럼명, 급수) 제외 + 즉시 재계산
- 제외 바에 칩으로 표시, 개별/일괄 해제 가능
- localStorage 저장

### 7. 결과 표시
각 그룹 카드:
- 추천 엠블럼 4개 (이미지 썸네일 + 이름 + 급수 + 태그)
- 태그: `고유` / `×2` (중복) / `✓ 보유` / `🔧 급수 업그레이드` / `추가 필요` / `🆕 신규`
- `(최종 ≈ +N)` 실제 최종 스탯 증가량 환산 표시
- **합산 옵션** 섹션
- **교체 권장** (제거/추가/유지)
- **현재→추천 변화량** (카테고리별 Δ)

### 8. 🎯 교체 시 최종 스탯 예측 (신규)
- 그룹 결과와 전체 목록 사이
- 모든 카테고리의 현재 최종 → 새 최종 → Δ 표시
- 플랫: `새최종 = 현재최종 + (추천기여 − 현재기여)`
- %: `새% = 현재% − 현재기여 + 추천기여`

### 9. 전체 엠블럼 목록
- 그룹별 접힘 테이블 (급수별 수치를 컬럼으로)
- 현재 유효 급수는 금색 하이라이트
- 이름 / 고유 / 능력치 / 1급~5급 / 최대급수

## 🧮 스코어링 로직

```
score = tierWeight(stat1) × relImprovement(v1) + tierWeight(stat2) × relImprovement(v2)
```

- **tierWeight**: [100, 85, 70, 58, 47, 38, 30, 24, 19, 15, 12, 9, 7, 5, 4, 3, 2, 1]
- **relImprovement**:
  - 플랫 (공격 +400): `v / baseStat` (기본값 대비 비율)
  - 보너스 (공격보너스 +7.2%): `v / currentBonus%` (현재 보너스 대비 비율)
  - 피해감소: `v / (100 − 현재%)` (감소율 개선)
  - 피해보너스: `v / (100 + 현재%)` (배율 개선)

**정렬**: 1차 티어 합 → 2차 실효 증가량 → 3차 급수 → 4차 고유
